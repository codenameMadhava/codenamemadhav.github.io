Overall Architecture

The architecture follows a clean separation-style backend flow:

  • User queries the app via REST API
  • Flask handles the HTTP request and routing
  • Pandas processes CSV data and computes analytics
  • Psutil performs real-time system resource monitoring
  • Computed metrics are returned as JSON/HTML responses
  • Dashboard visualizes data with interactive cards and charts
Client ‚Üí Flask API ‚Üí Pandas ‚Üí CSV Data ‚Üí Metrics ‚Üí System Monitor ‚Üí Dashboard UI

Key Libraries & Why They Are Used

1. Flask

Flask is used as a lightweight web framework to expose REST API endpoints. It handles HTTP requests, JSON payloads, routing, and responses.

  • Expose /health endpoint for service monitoring
  • Expose /summary endpoint for aggregated stats
  • Expose /devices and /activities for data queries
  • Act as the API layer between client and analytics system

Purpose: Lightweight web framework for building REST APIs and serving HTML templates.

2. Pandas

Pandas is used for efficient data manipulation and analysis of CSV datasets from Apple Watch, Fitbit, and Weka sources.

  • Load and parse CSV files efficiently
  • Perform aggregations (mean, sum, min, max)
  • Handle missing data and data type conversions
  • Group and filter health metrics by device type
  • Use the DataFrame model for efficient data processing

Purpose: Data manipulation, aggregation, and statistical analysis of health datasets.

3. Psutil

Psutil is used for cross-platform system and process monitoring. It provides real-time metrics for CPU, memory, disk, and network usage.

  • Monitor CPU usage percentage in real-time
  • Track memory consumption (used/total/percentage)
  • Monitor disk space utilization
  • Retrieve system cache information
  • Perform fast nearest neighbor search

Purpose: Real-time system resource monitoring and performance metrics collection.

4. Docker

Docker is used to containerize the entire application, including Python runtime, dependencies, and system libraries.

  • Ensure consistent runtime across environments
  • Simplify deployment with single container image
  • Eliminate environment dependency issues
  • Enable DevOps-style deployment workflow

Purpose: Containerization for consistent deployment across development, staging, and production environments.

Core Code Workflow

Data Loading

CSV datasets are loaded from the data/ folder, cleaned, and converted into Pandas DataFrames at application startup. These DataFrames are cached in-memory for fast retrieval.

Metrics Computation

  • Data is aggregated using Pandas groupby and agg functions
  • Statistical metrics (mean, sum, min, max) are computed
  • System metrics are collected via psutil in real-time
  • Docker container status is checked using Docker API
  • All computed metrics are returned as JSON responses

Query Handling

The query text is converted into an embedding using Sentence Transformers. FAISS searches the nearest document vectors and retrieves the top K results. The relevant documents are returned as contextual responses.

REST API Design

The API follows RESTful conventions with clear endpoint naming and JSON output, making it easy to integrate with other systems or frontends.

GET /health # Health check endpoint GET /summary # Aggregated statistics GET /devices # Device information GET /activities # Activity breakdown GET /api/dashboard # Dashboard metrics GET /api/system # System resource metrics
